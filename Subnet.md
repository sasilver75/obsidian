> Think of it like a hospital with different floors/wings for different types of patients; Maternity ward on one flor, Security on another. We do the same with our network; Our public-facing frontend servers might go in Subnet 1 with IP address range 10.0.1.x, our Application servers might go in Subnet 2 with [[Internet Protocol|IP Address]] 10.0.2.x, and our Database servers might go in subnet 3 with 10.0.3.x. 
> If they're in separate subnets, how does the Application server talk to the Database? Through [[Routing]]! Figures out how to get from Point A to Point B.
> Just because we *can* route traffic between subnets doesn't mean we should! This is where [[Network Firewall|Firewall]]s become necessary, specifically a [[Host Firewall]], which protects individual servers ("Only accept connections on Port 3306, and only from IP addresses in the frontend subnet! Anything else gets dropped!"). Also [[Network Firewall]]s which sit between subnets and filter traffic. We might place one between the public internet and our frontend subnet that says "Allow incoming traffic on port 80 and port 443, but block everything else!" It's a layered approach.
> We can't give each server a public IP address (costs money, need to manage them, and we'd need 50 of them). That's where [[Network Address Translation]] (NAT) comes in, which lets multiple devices with private IP addresses share one public IP address when accessing the internet. When backend server 10.0.2.5 wants to reach an external site, it sends a request to a NAT device, which replaces the private source address with its own public address. When the response comes back, the NAT device remembers that this message is destined for 10.0.2.5, and forwards it. Think of it as a receptionist in an office.
> Moving to the cloud... where we're renting computing resources instead of owning them. We still have IP addresses, ports, subnets, routing, firewalls, and NAT. The Cloud just provides some of these as managed services. We create a [[Virtual Private Cloud]], or VPC. Think of it as a floor in an office building; no one can enter your floor, even though they're in the same building. We create [[Subnet]]s, as before.
