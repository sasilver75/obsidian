Data Models

Raster vs Vector

🟦 Raster

A continuous field represented as a grid of cells (pixels). Each cell holds a numeric value.

  • Satellite imagery (pixels = color values)
  • Elevation models (pixels = meters ASL)
  • Temperature grids, land cover maps
GeoTIFF COG Zarr
📐 Vector

Discrete geometric shapes with attributes attached. Three types:

  • Point: crime incident, tree, 311 call
  • Line: street segment, river, route
  • Polygon: neighborhood, parcel, park
GeoJSON GeoPackage FlatGeobuf
Vector answers "Where is this thing?" with precise boundaries. Raster answers "What value exists at every point?" with continuous fields.
Data Formats

Vector Formats

Format Best For Key Gotcha
Shapefile (.shp) Legacy compatibility, government downloads Split across 3+ files, 10-char column names, 2GB limit
GeoJSON Web APIs, small datasets, config boundaries Verbose, no spatial index, WGS84 only, 50MB+ for cities
GeoPackage (.gpkg) Local storage, replacing Shapefiles Not suited for concurrent writes or cloud-native access
FlatGeobuf (.fgb) Serving from S3/CDN via range requests Spatial index must be at file start; binary, not human-readable
GeoParquet Analytics, large datasets, DuckDB/Spark Columnar; needs tooling (not a single-file viewer like QGIS)
Avoid Shapefiles when you have a choice. GeoJSON for small web data, GeoPackage for local exchange, GeoParquet for analytics.
Coordinate Systems

Coordinate Reference Systems

Latitude = North/South, –90 to 90 (negative = south of equator)
Longitude = East/West, –180 to 180 (negative = west of Greenwich)

⚠️ Gotcha: GeoJSON, Mapbox, and most web APIs use (lon, lat) order — not lat/lon! Many bugs come from swapping these.

Common EPSG Codes

  • EPSG:4326 (WGS84) — Storing coordinates, GPS output, GeoJSON
  • EPSG:3857 (Web Mercator) — Displaying tiles in web maps
  • Local projected CRS — Accurate distance/area calculations

Two Coordinate Types

Geodetic (EPSG:4326)

Position expressed as latitude, longitude, height. Origin is the ellipsoid surface. What GPS gives you. Note: geodetic latitude ≠ geocentric latitude due to Earth's flattened poles.

Geocentric (XYZ)

Origin is Earth's center of mass. Position as (X, Y, Z) in a Cartesian system. Used in satellite orbital mechanics.

Projections

Map Projections

The Earth is an oblate spheroid. A projection is the mathematical transformation to a flat map. All projections distort something.

Conformal

Preserves local angles and shapes. Good for navigation.

Web MercatorUTM

Distorts: area (Greenland looks bigger than Africa)

Equal-Area

Preserves area. Good for statistical analysis and comparisons.

Albers Equal Area

Distorts: shape and angles

Equidistant

Preserves distance from a specific point. Good for range/coverage analysis.

Azimuthal Equidistant

Distorts: area and shape elsewhere

Rule: Store in EPSG:4326 (universal, what every API gives you). Display on web maps in EPSG:3857 (tile renderer handles conversion). Calculate distances/areas in a local projected CRS or use PostGIS's geography type.
Earth Observation

Remote Sensing

Acquiring information about the Earth without making physical contact — using satellite or airborne sensors to detect and classify objects.

⚡ Active Sensing

Sensor emits a signal and measures the return. Works day or night, through clouds.

  • SAR (RADAR): Microwave pulses, measures backscatter
  • LiDAR: Laser pulses, measures time-of-flight
  • Altimeter: Measures height above surface
☀️ Passive Sensing

Sensor detects reflected sunlight or emitted radiation. Cannot penetrate clouds.

  • Optical: Visible and near-infrared bands
  • Thermal IR: Detects heat emitted by surfaces
  • Hyperspectral: Hundreds of narrow wavelength bands
Fundamental principle: All objects above absolute zero emit electromagnetic radiation. The amount reflected depends on surface roughness and albedo (reflectivity).
Remote Sensing

Four Types of Resolution

📏 Spatial Resolution

The size of one pixel on the ground. Ranges from <30cm (Maxar commercial) to 1km+ (weather satellites).

🌈 Spectral Resolution

The number and width of wavelength bands captured. RGB = 3 bands; Landsat = 7+ bands; Hyperspectral = 200+ bands.

💡 Radiometric Resolution

The number of intensity levels the sensor can distinguish. Typically 8–14 bits (256 to 16,384 shades) per band.

🕐 Temporal Resolution

How often the satellite revisits the same location. Ranges from minutes (geostationary) to weeks (high-res commercial).

There's always a tradeoff: higher spatial resolution usually means narrower swath width, lower revisit frequency, and higher cost.
Remote Sensing

Data Processing Levels

A taxonomy defined by NASA in 1986, now widely adopted:

LevelDescriptionUsability
L0Raw, unprocessed instrument data; communication artifacts removedSpecialist only
L1aL0 + calibration coefficients appended (not applied). The fundamental data record.Specialist only
L1bL1a processed to sensor units (e.g. radar backscatter, brightness temperature)Advanced users
L2Derived geophysical variables (ocean wave height, soil moisture). First directly usable level.✅ Most science
L3Variables mapped onto uniform space-time grids; missing data interpolated, mosaicked✅ Most applications
L4Model outputs or analyses derived from lower-level data (not directly measured)✅ End products
L2 = Top of Atmosphere (ToA) — geometrically corrected, geolocated, but with atmospheric haze.
L2 = Surface Reflectance (SR) — atmospheric correction applied; better for surface analysis, worse if studying the atmosphere.
Spectral Analysis

Spectral Indices

Mathematical combinations of spectral bands that highlight specific surface properties.

🌿 NDVI — Vegetation
(NIR − Red) / (NIR + Red)
  • 0.6–0.9: Dense healthy vegetation
  • 0.2–0.5: Sparse / grassland
  • −0.1–0.1: Rock, sand, snow, dead vegetation
  • Negative: Water, clouds

The NDVI curve over time distinguishes crop types — corn peaks at a different time than soy.

Other Common Indices
  • NDWI: (Green − NIR) / (Green + NIR) — water bodies
  • NDBI: (SWIR − NIR) / (SWIR + NIR) — built-up areas
  • NBR: (NIR − SWIR) / (NIR + SWIR) — burn severity
  • NDRE: Red-edge band — crop stress, nitrogen content
  • LST: Land Surface Temperature from thermal bands
NDVI saturates in dense canopies and is sensitive to atmospheric conditions and soil background. The red-edge band (Sentinel-2 has it; Landsat doesn't) significantly improves vegetation analysis.
Active Remote Sensing

Synthetic Aperture Radar (SAR)

An active radar system that emits microwave pulses and measures backscattered energy. Works day or night, through clouds and smoke.

Why SAR Matters

  • Weather-independent — microwaves penetrate clouds
  • Sensitive to surface roughness and moisture
  • Can detect millimeter-scale surface deformation (InSAR)
  • Penetrates vegetation; L-band reaches soil beneath canopy

Radar Bands

X-band High resolution, Sentinel-1 alt.

C-band Sentinel-1, Radarsat

L-band Penetrates vegetation, soil

P-band Deepest penetration, biomass

Polarization

Each pulse can be sent and received in horizontal (H) or vertical (V) polarization.

  • VV: Best for ocean surface, urban
  • VH: Best for vegetation volume scattering
  • HH: Penetrates into vegetation/soil
  • Quad-pol: All 4; richest info but larger data
InSAR: Interfering two SAR acquisitions to measure surface deformation at mm precision. Used for earthquake monitoring, subsidence, volcanic activity.
Active Remote Sensing

LiDAR

Emits laser pulses and measures time-of-flight to build a 3D point cloud — millions of XYZ coordinates with intensity values.

Key Superpower: Multiple Returns

  • One laser pulse can generate 2–5+ returns
  • First return: Top of tree canopy → DSM
  • Last return: Ground through vegetation gaps → DTM
  • DSM − DTM = Canopy Height Model (CHM)
No other technology reliably sees through vegetation to the ground at scale. This makes LiDAR irreplaceable for forestry and bare-earth terrain.

ASPRS Classification Codes

Class 2:  Ground
Class 3:  Low vegetation (0–0.5m)
Class 4:  Medium vegetation
Class 5:  High vegetation (>2m)
Class 6:  Building
Class 9:  Water
Class 17: Bridge deck

Point Density

  • 1 pt/m²: Decent DEM
  • 8 pts/m²: Reliable building extraction
  • 25+ pts/m²: Individual tree measurement
Terrain Data

Elevation Models

DEM

Digital Elevation Model — generic term for any elevation dataset. Can refer to either DSM or DTM depending on context.

DSM

Digital Surface Model — Includes everything on the surface: buildings, trees, vehicles. First LiDAR return.

DTM

Digital Terrain Model — Bare earth only; above-ground objects filtered out. Last LiDAR return.

How Elevation Models Are Created

MethodHowAccuracy
LiDARActive laser ranging; last return = groundBest (±5cm)
InSARPhase difference between two SAR passes (SRTM)~1–10m
StereophotogrammetryFeature matching from two optical images at different angles~1–5m
Structure from Motion (SfM)Many overlapping drone images; common for site surveys~2–10cm
SRTM (Shuttle Radar Topography Mission, 2000) is the most widely used free global DEM at 30m resolution, created via InSAR.
Applications

Common Remote Sensing Tasks

🔄 Change Detection

Detect what changed between two observations. Band differencing, NDVI delta, SAR coherence. Challenge: separating real change from seasonal variation and atmospheric noise.

🗺 Land Cover Classification

Assign every pixel to a class (forest, water, urban, cropland). Note: land cover (what's there) ≠ land use (what it's used for).

🏗 Object Detection

Detect and delineate specific objects: buildings, ships, vehicles, solar panels. Deep learning dominant now. Challenge: tiny objects, sparse labels, geographic generalization.

📈 Time-Series Analysis

Track how pixel values change over time. Phenology (crop growth cycles), deforestation trends, drought stress. Challenge: gaps from clouds and atmospheric noise.

Every remote sensing product needs validation — confusion matrices, per-class precision/recall. A product without rigorous accuracy assessment is scientifically incomplete.
Web Mapping

Web Map Tiles

Web maps divide the world into a pyramid of square tiles, each addressed by Z/X/Y — zoom level, column, row.

Zoom Level Scale
  • Zoom 0: Entire world in 1 tile
  • Zoom 1: 4 tiles (doubles in each dimension)
  • Zoom 15: ~1 billion tiles (street level)
  • Zoom 20: Building-level detail

Each zoom level quadruples the tile count. ~22 zoom levels cover all practical use cases.

🖼 Raster Tiles

Pre-rendered PNG/JPEG images. Simple to serve via CDN. Used for satellite imagery basemaps. Not interactive — can't change style or query features.

⚡ Vector Tiles (.pbf)

Binary geometry + properties. Rendering happens in the browser. Styles can change, features are queryable, tiles are smaller. Used by MapLibre GL JS and Mapbox GL JS.

Tools: pg_tileserv (Go, from PostGIS on-demand) · Martin (Rust, high performance) · TiTiler (Python, raster tiles from COGs)
Architecture

Cloud-Native Geospatial

A philosophy for storing and accessing geospatial data designed from the ground up for cloud object storage — not adapted from legacy GIS workflows.

The shift is analogous to web apps moving from server-rendered pages to APIs + client-side rendering. Analysis that required downloading TBs can now run against data in place.
📁 COG

Cloud-Optimized GeoTIFF — rasters reorganized for HTTP range requests. Serves map tiles without loading the full file.

📊 GeoParquet

Columnar vector format built on Apache Parquet. Query massive datasets with DuckDB or Spark without downloading everything.

🗂 STAC

SpatioTemporal Asset Catalog — standardized metadata for discovering geospatial assets. The map to your data.

Standards stewarded by the Cloud-Native Geospatial Foundation (Linux Foundation project). Members include AWS, Microsoft, Google, ESRI, Planet, Maxar.
Cloud-Native Formats

Cloud-Optimized GeoTIFF (COG)

The Problem with Classic GeoTIFF

Pixels stored row-by-row. To serve a map tile, you'd need to decompress the entire file to extract relevant pixels. Unusable for on-demand tile serving.

COG Internal Structure

  • Index at the start: Small header listing where every tile and overview lives
  • Tiled internally: Pixels stored in 256×256 or 512×512 squares
  • Overview pyramids: Pre-downsampled versions at multiple zoom levels, all inside one file
Result: A 5GB satellite image on S3 can serve web map tiles by fetching only the bytes needed — without ever loading the full image into memory.
HTTP Range Request

Fetch the header → find where zoom-level-8 tiles are → issue a range request for exactly those bytes → return the 256×256 tile. No full download needed.

STAC + COG is the standard stack for satellite imagery: STAC tells you what exists and where, COG lets you efficiently read only the spatial subset you need.
Metadata Standard

STAC — SpatioTemporal Asset Catalog

An open specification for describing geospatial assets in a consistent, searchable way. STAC is metadata, not data. It is the map to your data; your map and data don't need to live in the same place.

4 Components
  • Catalog: Top-level container (file folder)
  • Collection: Groups related items + temporal/spatial context
  • Item: One scene/asset; a GeoJSON feature with STAC fields
  • Assets: Actual files (COGs, Zarr stores) that the Item points to
Static vs Dynamic Catalogs

Static: JSON blobs in object storage. Simple, no servers needed. Great for full scans; bad for search.

Dynamic (API): Database-backed. Filter by bbox, datetime, cloud cover. Bad for full scans; great for search.

Key pattern: Search API returns Items; Items point to COG assets in S3; your code issues HTTP range requests directly to those COGs. No intermediate server required.
Spatial Indexing

H3 — Hierarchical Hexagonal Indexing

Why Hexagons?

  • Exactly 6 equidistant neighbors (squares have 8, diagonals are further)
  • Consistent for spatial smoothing and neighbor queries
  • No gaps or overlaps; look better on maps

Key Properties

  • Hierarchical: Each res-8 cell contains exactly 7 res-9 cells
  • Universal: Any lat/lon → H3 index is O(1), pure function
  • Parent lookup = a bitmask operation, not a table lookup
H3 has 12 pentagonal cells (icosahedron artifact) with only 5 neighbors — watch for edge cases in spatial algorithms.

Resolution Reference

Res 6  ≈ 36 km²   city scale
Res 7  ≈ 5.2 km²  large neighborhood
Res 8  ≈ 0.7 km²  few city blocks ← sweet spot
Res 9  ≈ 0.1 km²  one block face
Res 10 ≈ 0.015 km² half a block
Res 12 ≈ 0.0003 km² building scale
Typical Usage Pattern

H3 column + B-Tree index for "paint the city" aggregation views.

geometry column + GiST index for "what's near this specific point" detail queries.

They're complementary — use both.

Summary

The Mental Model

Store & Exchange
  • WGS84 (EPSG:4326) for coordinates at rest
  • GeoJSON for small vector data, APIs
  • GeoParquet for large vector analytics
  • COG for raster in the cloud
  • STAC for discovering any of the above
Analyze & Compute
  • Project locally before distance/area calculations
  • NDVI + spectral indices for vegetation, water, burn
  • SAR when you need cloud-penetrating or night imagery
  • LiDAR last-return for bare-earth DTM
  • H3 res 8–10 for spatial aggregation analytics
The challenge today is not data access — it's translation into insight. There are ~100,000 new Sentinel-2 images every week alone.
