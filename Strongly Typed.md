Compare with: [[Weakly Typed]]

A **strongly typed** language enforces type rules strictly: it does not silently coerce values between unrelated types, and operations that don't make sense for a given type produce an error rather than a surprising result.

Examples: Python, Java, Rust, Haskell. Adding a string to an integer raises an error rather than guessing what the programmer meant.

"Strong" vs. "weak" typing is a spectrum, not a binary, and is distinct from static vs. dynamic typing — Python is dynamically typed but strongly typed, while C is statically typed but relatively weakly typed (pointer casts, implicit numeric conversions).
