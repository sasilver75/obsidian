Alembic is a lightweight database migration tool in Python that's used with the [[SQLAlchemy]] database toolkit.

It tracks which migrations have been applied in a table called `alembic_version` that it creates automatically in your database, which contains a single row, the ==revision ID== of hte last applied ==migration==.

A ==migration== is an application of a ==revision==, which is a file with an upgrade/downgrade set of SQL statements that are executed. This creates something like a doubly-linked list of database versions that can be traversed by upgrading/downgrading using alembic.

Use `alembic upgrade head` to walk the chain from the `alembic_version` revision to the latest revision, applying each revision's `upgrade()` function in order.

`--autogenerate` is a diff tool, but is not a perfect code generator.
- It's good at new tables, dropped tables, added/removed columns, changed column types (sometimes), new/removed indexes
- It's not always great at *renamed* tables/columns (seeing them as drop+add), changes to stored procedures or views, custom PostgreSQL types, [[GeoAlchemy2]] geometry columns.

Always inspect autogenerated migrations before running `upgrade head`!
